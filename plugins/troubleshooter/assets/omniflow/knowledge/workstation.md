# OmniFlow Workstation Knowledge

## Use For

- goods-to-person picking station symptoms
- workstation lights, sensors, operator prompts, or task state
- robot entering/leaving workstation points

## Common Fault Branches

- workstation task state stuck or delayed
- light/sensor controller fault
- operator action not reflected in software state
- robot blocked near station but root cause is scheduler/motion, not station hardware
- tote-present signal mismatch

## Guardrail

A title mentioning `WS` or workstation is location evidence only. Confirm whether the root cause is station hardware, scheduler state, robot motion, or physical obstruction.
